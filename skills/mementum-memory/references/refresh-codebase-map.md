# Refresh Codebase Map

Use this after changes to top-level structure, entry points, packaging, tests, deployment, or architecture boundaries.

Inspect enough of the repository to update only high-value orientation facts:

1. top-level structure
2. important entry points
3. package/build configuration
4. test layout
5. scripts and deployment configuration
6. generated or do-not-edit areas
7. important architecture boundaries

Do not:

- list every file
- summarize every module/class/function
- copy large sections from config or README files
- invent architecture
- store secrets, runtime data, raw logs, private data, or large generated artifacts

For uncertain items, mark them as "To be verified."

After editing, summarize:

1. What changed.
2. Source pointers used.
3. Any assumptions or uncertainties.
